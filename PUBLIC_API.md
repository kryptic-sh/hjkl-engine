## Status

**Pre-freeze refresh.** 0.0.39 was the planned 0.1.0 cut; on entry the four
prior keystone patches (0.0.35–0.0.38) had moved search / marks / spans / fold
mutation off `Buffer`, but the `Editor` generic flip (`Editor<'a, B: Buffer,
H: Host>`) still hits residual concrete `hjkl_buffer::Buffer` reaches inside
`Editor::set_content` (`Buffer::from_str`), the default `Settings::wrap` field
(`hjkl_buffer::Wrap`), and ~143 `self.buffer.…` call sites in `vim.rs` whose
signatures take `&mut Editor<'_>`. Per the design doc's stop threshold, the
flip is deferred to 0.0.40 / 0.1.0 and 0.0.39 ships with `Query::dirty_gen`
landed (one-method trait expansion, last item gating the flip).

The freeze contract still lands at 0.1.0; this file regenerates each release
until then. Diff against the 0.0.33 baseline tracks the trait-extraction arc.

    cargo +nightly public-api -p hjkl-engine --simplified > crates/hjkl-engine/PUBLIC_API.md.raw
